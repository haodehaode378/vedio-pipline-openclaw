# ShopMotion v1.1 - 课程设计报告大纲

## 摘要

本项目设计并实现了一个电商带货短视频自动生成系统 ShopMotion，基于 React + TypeScript + Node.js + Remotion 技术栈，通过"商品输入 → 智能模板推荐 → 分镜脚本生成 → 视频渲染"的流程，设计了电商短视频的自动化生产方案。系统包含 15 个模板家族、45 个模板配置、20 个商品类别适配、30 个 Demo 商品和完整的 API 接口。当前版本为 v1.1 静态增强版，核心功能代码已编写完成（静态代码层面），运行验证待迁移到高配置环境后执行。

**关键词**：电商短视频、自动化生成、Remotion、模板系统、智能推荐

---

## 1. 项目背景

### 1.1 行业背景
- 短视频电商已成为主流营销方式
- 中小商家面临视频制作成本高、效率低的痛点
- AI 技术为内容自动化生产提供了新可能

### 1.2 项目动机
- 降低电商短视频制作门槛
- 探索模板化 + 智能化的视频生成方案
- 作为课程设计综合实践项目

### 1.3 项目目标
- 建立可扩展的 Remotion 模板体系
- 实现商品到短视频的自动化流程
- 支持多类别、多风格的视频生成

---

## 2. 需求分析

### 2.1 功能需求
1. 商品信息输入（名称、类别、价格、卖点等）
2. 智能模板推荐（根据商品类别和营销目标）
3. 分镜脚本自动生成（8种场景类型）
4. 字幕时间轴生成（SRT/ASS/JSON 导出）
5. 旁白时间轴生成（Mock TTS）
6. Remotion 视频渲染（预留接口）
7. 飞书通知集成（任务状态推送）
8. Demo 商品一键体验

### 2.2 非功能需求
1. 模板系统可扩展性（15→405+组合）
2. 商品类别覆盖度（20个大类）
3. 代码模块化和可维护性
4. 文档完整性

### 2.3 约束条件
- 当前服务器配置较低，不能运行真实渲染
- 不接入真实 LLM/TTS API（使用 Mock）
- 不使用真实品牌商标

## 3. 系统总体设计

### 3.1 系统架构
```
用户 → 前端(React) → API(Express) → 核心模块
                                      ├── 模板系统
                                      ├── 分镜生成
                                      ├── 字幕系统
                                      ├── 旁白系统
                                      └── 通知系统
```

### 3.2 模块划分
1. **product**：商品类型定义
2. **templates**：模板系统（家族、配置、注册表、解析器）
3. **storyboard**：分镜脚本类型和生成
4. **generation**：Mock LLM 生成
5. **captions**：字幕生成和导出
6. **voiceover**：旁白生成
7. **jobs**：任务管理
8. **notification**：飞书通知
9. **frontend**：React 前端
10. **server**：Express 后端

## 5. 功能模块设计

### 5.1 模板系统
- 三层架构：Family → Variant → Config
- 15 个模板家族，每个家族 3 个变体
- 45 个模板配置，支持 405+ 组合
- 智能推荐算法（类别50% + 目标30% + 风格20%）

### 5.2 分镜生成
- 8 种场景类型：hook、product_reveal、selling_point、scenario、comparison、proof、price_offer、cta
- 根据商品类别差异化生成
- 支持场景意图、布局提示、模板插槽

### 5.3 字幕系统
- 支持 SRT、ASS、JSON 三种格式
- 中文断句规则
- 字幕密度自适应（15s/30s/45s）

### 5.4 旁白系统
- 语速标记（slow/normal/fast）
- 情感标记（neutral/excited/warm/urgent）
- 与场景时长对齐

---

## 6. 数据结构设计

### 6.1 商品输入（ProductInput）
- 14 个字段，覆盖商品基本信息和视频偏好

### 6.2 模板家族（TemplateFamily）
- 19 个字段，包含视觉配置和推荐信息

### 6.3 模板配置（TemplateConfig）
- 22 个字段，完整的渲染配置

### 6.4 分镜脚本（Storyboard）
- 包含视频元数据、商品信息、场景列表、字幕、旁白、渲染配置

### 6.5 场景（Scene）
- 15 个字段，包含场景意图、布局提示、模板插槽

---

## 7. 模板推荐算法

### 7.1 算法概述
基于多维度加权评分的模板推荐算法。

### 7.2 评分维度
1. **类别匹配**（权重 50）：商品类别与模板家族的 suitableCategories 匹配
2. **营销目标匹配**（权重 30）：营销目标与模板家族的 suitableGoals 匹配
3. **风格匹配**（权重 20）：用户选择的视频风格与模板家族匹配

### 7.3 回退机制
- 无匹配时使用 WarmLifestyle 作为默认
- 置信度从 0.85 降至 0.3

---

## 8. Storyboard 生成流程

### 8.1 生成步骤
1. 接收商品输入和模板推荐
2. 根据类别选择分镜结构
3. 生成 Hook 场景（痛点引入）
4. 生成 Product Reveal 场景
5. 根据类别生成中间场景（卖点、场景、对比、证据）
6. 生成 CTA 场景
7. 同步生成字幕和旁白时间轴

### 8.2 类别差异化
- 美妆类：成分讲解 + 使用对比
- 母婴类：安全材质 + 温馨场景
- 食品类：食欲场景 + 口味卖点
- 数码类：参数对比 + 场景演示
- 家居类：使用前后 + 空间感

---

## 9. Remotion 模板系统

### 9.1 共享组件
- ProductNameBlock：商品名称展示
- SellingPointCard：卖点卡片（4种样式）
- CaptionBlock：字幕显示（4种布局）
- CTABlock：CTA 按钮（4种动效）
- PriceBlock：价格展示
- ProgressIndicator：进度指示器
- ProductImagePlaceholder：商品图片占位

### 9.2 配置驱动
所有组件通过 TemplateConfig 配置驱动，不同模板家族通过不同配置实现视觉差异。

## 10. 字幕与旁白时间轴

### 10.1 字幕时间轴
- 每个场景对应一个字幕片段
- 支持断句和换行
- 支持高亮和关键词样式

### 10.2 旁白时间轴
- 旁白时长与场景时长对齐
- 支持语速和情感标记
